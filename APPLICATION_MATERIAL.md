# FlowDesk AI Ops Hub 项目申请材料草稿

## 项目名称

FlowDesk AI Ops Hub：多行业知识库工单处置 Agent

## 项目背景

大量企业的一线运营场景存在共同痛点：SOP 文档分散、历史经验难复用、工单描述不规范、故障处置依赖老员工经验。门店、客服、校园后勤、设备巡检等团队每天都会遇到重复性问题，但解决过程缺少统一的知识入口和可审计沉淀。

本项目希望用大模型 + RAG + 工单系统构建一个轻量、可落地、可扩展的 AI 处置助手。用户输入现场问题后，系统会自动检索相关 SOP 和 FAQ，判断风险等级，生成处理步骤，并把问答过程沉淀为工单数据。

## 目标用户

- 连锁门店运营人员
- 电商客服团队
- 校园/园区后勤人员
- 设备巡检与运维人员
- 中小企业内部 IT / 行政支持团队

## 核心功能

1. **知识库检索增强问答**
   - 从 SOP、FAQ、故障手册中检索相关片段
   - 输出答案时附带引用来源，便于审核和追责

2. **工单自动分级**
   - 根据问题描述识别紧急程度
   - 给出建议责任团队与 SLA

3. **处置步骤生成**
   - 生成一线人员可直接执行的 checklist
   - 区分临时止损、根因排查、升级条件

4. **多模型适配**
   - 不绑定单一模型厂商
   - 支持 OpenAI-compatible 网关、Ollama、本地模型和自定义企业模型服务

5. **数据沉淀**
   - 保存工单标题、描述、AI 建议、状态和创建时间
   - 后续可用于质检、复盘和模型评测

## 技术实现

- FastAPI 提供后端接口
- SQLite 存储工单和问答记录
- 轻量 TF-IDF 检索实现 RAG 原型
- 多 Provider 模型适配层封装模型调用
- 原生 Web 页面用于演示和业务验证
- pytest 覆盖 RAG、分级和 API 基础逻辑

## 落地价值

- 降低一线人员查 SOP 的时间
- 提升工单描述规范度
- 让新员工也能按照标准步骤处理问题
- 保留 AI 处置依据，降低幻觉风险
- 方便后续接入真实企业系统
